Sessions / SS (也有人叫Visits)

Sessions (Visits) 工作階段

The period of time a user is active on your site or app.

超過 30 分鐘，視為新的工作階段。如果離開網站不到 30 分鐘就回來，仍算在原本的工作階段。

User vs Session

一個 User 在一定的時間內 (Day / Week / Month) 可能有多次的 Session，
內容網站 Session / User 越大越好，通常是回頭客，可以只算回頭客的 SS/UU，內容網站 Time / Session 通常越大越好

Session 是複雜的組合

End of day expiry

換日自動換 Session
• opens your website at 11:50 PM and leaves your website at 12:10 AM
• The first session ends at 11:59:59 PM, and the second session begins at 12:00 AM.
所以 Session 數會被高估，特別在晚上

App SDK 可以自行設定新 Session

常見觸發新 Session 的方法

• 手機通知
• 產品改善
• 製造使用者焦慮
• 讓使用者儲存數位社交資產

Session Duration Visit Duration

Pageviews (PV)

Pageview Definition

A pageview (or pageview hit, page tracking hit) is an instance of a page being loaded (or reloaded) in a browser. Pageviews is a metric defined as the total number of pages viewed.

Pageview 計算

與 CPM 廣告相關，經常被當成最重要指標
Pageviews: 看多少次就算多少次
只看這個數值必然走火入魔
Unique pageview: 在同一個 Session 內看 多少次都算一次

PV 注意事項

Frame 架構下(現在少見)，PV會高估
不計較廣告收入的網頁通常會減少 (ex.報導者)
Facebook 這樣的長網頁不計較頁數

Bounce and Exit

Bounce(跳出) vs Exit (離開)

Bounce % = 有多少使用者只看一頁，作為第一頁就離開的比例
Exit % = 有多少頁面是「最後一頁」

Bounce Rate

只有 1 PV 的 Session 佔全部 Session 的比例
內容網站可能高達90%
與 Session Duration 呈反比

Exit Page

離開頁
可以優化、導流
內容網站 Landing Page 與 Exit Page 經常 一樣
電子商務網站會不同

Exit %

Content Page 應該都差不多 Exit %
Index Page 應該追求極低 Exit %
明確目的 der Flow 中間接段都應該低 Exit %

Bounce 跳出 vs Exit

• Bounce
-- 只看一頁
-- Session Time:0
• Exit
-- 離開頁
-- 開最後一頁時算 Session 結束
-- 這一頁不算時間
